Git基础操作教程

发表于 2020-05-24 分类于 Git 阅读次数：
本文字数： 1.8k 阅读时长 ≈ 2 分钟

1 Git 是什么

Git是一个版本管理控制系统（缩写VCS），它可以在任何时间点，将文档的状态作为更新记录保存起来，也可以在任何时间点，将更新记录恢复回来。

2 Git下载安装

下载地址： https://git-scm.com/downloads

在安装的过程中，所有选项使用默认值即可。安装完成后，在桌面点击鼠标右键，出现如下图两个按钮即为安装成功。

  • Git GUI Here 是图形化界面的 Git 工具
  • Git Bash Here 才是我们需要的命令行工具

3 Git 基本工作流程

git仓库 暂存区 工作目录
用于存放提交记录 临时存放被修改文件 被Git管理的项目目录

4 Git 的使用

4.1 Git 使用前配置

在使用 git 前，需要告诉 git 你是谁，在向 git 仓库中提交时需要用到。

  1. 配置提交人姓名：git config --global user.name 提交人姓名
  2. 配置提交人姓名：git config --global user.email 提交人邮箱
  3. 查看git配置信息：git config --list

注意

  1. 如果要对配置信息进行修改，重复上述命令即可。

  2. 配置只需要执行一次。

4.2 提交步骤
  1. git init 初始化git仓库
  2. git status 查看文件状态
  3. git add 文件列表 追踪文件
  4. git commit -m 提交信息 向仓库中提交代码
  5. git log 查看提交记录

注意

鼠标右击打开Git Bash Here，然后输入git init 初始化一个 Git 仓库后，它会在当前目录下生成一个.git的隐藏文件夹(要打开显示隐藏文件夹才能看到)。这个文件夹是 Git 来跟踪管理版本的，没事千万不要乱改这个目录里的文件，否则，很容易就照成仓库损坏。

4.3 撤销
  • 用暂存区中的文件覆盖工作目录中的文件： git checkout 文件
  • 将文件从暂存区中删除： git rm --cached 文件
  • 将 git 仓库中指定的更新记录恢复出来，并且覆盖暂存区和工作目录：git rest --hard commitID

5 Git分支

我们在开发大型项目的时候，往往会多人协作，每个人负责不同的版块的开发，这个时候，我们就需要用到 Git 工具的创建分支功能了。使用分支，可以让我们从开发主线上分离出来，以免影响开发主线。

5.1 分支细分
  1. 主分支（master）：第一次向 git 仓库中提交更新记录时自动产生的一个分支。
  2. 开发分支（develop）：作为开发的分支，基于 master 分支创建。
  3. 功能分支（feature）：作为开发具体功能的分支，基于开发分支创建
5.2 分支命令
  • git branch 查看分支
  • git branch 分支名称 创建分支
  • git checkout 分支名称 切换分支
  • git merge 来源分支 合并分支
  • git branch -d 分支名称 删除分支

注意：分支被合并后才允许删除，但也可通过git branch -D 分支名称 强制删除

6 暂时保存更改

在git中，可以暂时提取分支上所有的改动并存储，让开发人员得到一个干净的工作副本，临时转向其他工作。

使用场景：分支临时切换

  • 存储临时改动：git stash
  • 恢复改动：git stash pop

7 使用GitHub远程仓库

7.1 注册github账户

打开官网 https://github.com/ 首页，点击 Sign up 按照提示注册账号。

7.2 创建仓库

进入github个人中心，点击加号然后选择New repository创建一个仓库。

7.3 将本地仓库推送到远程仓库

如果你没勾选生成初始化README文档，那么在仓库的页面中，我们可以看到这样的信息

可以看到有三种方法将本地仓库推送到远程仓库。

由于git init git add git commit这三个命令我们已经执行过了。然后我们只要执行最后的两条命令即可将项目上传至Github上，直接复制下来，在 Git 窗口中粘贴执行就行了。

7.5 git推送远程仓库命令

  1. git push 远程仓库地址 分支名称

  2. git remote add 远程仓库地址别名 远程仓库地址

  3. git push 远程仓库地址别名 分支名称

  4. git push -u 远程仓库地址别名 分支名称

    -u 记住推送地址及分支，下次推送只需要输入git push即可

7.6 克隆仓库

克隆远端数据仓库到本地：git clone 仓库地址

7.7 拉取远程仓库中最新的版本

拉取远程仓库中最新的版本：git pull 远程仓库地址 分支名称

以上就是Git的基础操作内容啦，感谢观看~

高级的操作以及多人协作开发以后再更新吧，学会这些你就已经可以满足日常开发的需求啦~
